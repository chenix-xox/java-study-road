### 位序（Bit Ordering）

| 指数值   | 7    | 6    | 5    | 4    | 3    | 2    | 1    | 0    |
| -------- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 二进制值 | 0    | 1    | 0    | 0    | 1    | 0    | 1    | 0    |
| 十进制值 |      | 64   |      |      | 8    |      | 2    |      |

64 + 8 + 2 = 74

> 以上呢，是**正常二进制转十进制**的写法
>
> 但是实际上我们还可以写成...

| 指数值   | 0    | 1    | 2    | 3    | 4    | 5    | 6    | 7    |
| -------- | ---- | ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 二进制值 | 0    | 1    | 0    | 0    | 1    | 0    | 1    | 0    |
| 十进制值 |      | 2    |      |      | 16   |      | 64   |      |

2 + 16 + 64 = 82



> 计算机两种方法都可以识别
>
> 第一种我们经常使用的，从高位向低位计算的，称为**大端位序（Big Endian Bit Order）**
>
> 而第二种，从小位往高位的计算的，成为**小端位序（Little Endian Bit Order）**



位元组顺序（bit-endianness）



### 字节序（Byte Ordering）

> **字节序**： 字节数据在内存中的存储顺序

**如下`二 -> 十六`，默认以大端位序的方式转换**

| 二进制   | 0110 | 1011 | 0010 | 1101 |
| -------- | ---- | ---- | ---- | ---- |
| 十六进制 | 6    | B    | 2    | D    |

将**大端，从低位**依次放入内存，称为**大端字节序（大端序）**

将**小端，从低位**依次放入内存，称为**小端字节序（小端序）**
